# Day 5

## 솔루션 패키지 구성 및 운영 구조 검토

* 문제: 제품 배포 패키지에 포함되어야 하는 구성 요소와 운영 시 실수를 방지하기 위한 구조에 대한 검토

* 검토 대상

    * **솔루션 패키지 기본 구성**

        * 전체 모듈을 제어하는 공용 실행(CTL) 스크립트
        * 제품 실행 스크립트
        * 모듈별 실행 스크립트
        * 모듈별 설정(Configuration) 파일
        * 로그 디렉터리
        * 제품 운영에 필요한 공용 리소스

    * **내장 JDK 제공 방식**

        * 제품과 함께 자체 JDK를 포함하여 실행 환경을 표준화
        * 현재 어떤 JDK를 사용 중인지 운영자가 직관적으로 확인할 수 있는 장치 필요
        * 고객사 JDK를 사용할 경우 내장 JDK가 사용되지 않음을 명확하게 표시하는 구조 필요
        * 예시로 JDK 디렉터리명을 `disabled`로 변경하거나, 사용 여부를 나타내는 파일명을 변경하는 등의 운영 장치 검토

    * **공용 CTL(Command) 운영 방식**

        * 공용 CTL 적용 시 모듈별 `.rc` 파일 실행을 누락하면 다른 모듈 설정이 적용될 위험 존재
        * 운영자의 실수를 방지하기 위해 모듈별 CTL 명령어를 분리하는 방안 검토
        * 잘못된 모듈의 CTL을 실행한 경우 즉시 오류를 발생시켜 실행을 차단하는 Fail-fast 방식 적용 검토

* 결론: 패키지는 실행 파일뿐 아니라 운영자의 실수를 예방하는 구조까지 포함하는 것이 중요하며, 실행 환경(JDK)과 모듈별 실행 컨텍스트를 명확히 구분하여 오동작 가능성을 최소화하는 방향이 적절함

    * **내장 JDK 사용 여부를 명확히 표현**: 디렉터리명 또는 상태 파일을 통해 현재 사용 중인 JDK를 즉시 확인 가능하도록 구성
    * **모듈별 실행 환경 분리**: CTL 실행 시 반드시 해당 모듈의 환경 설정만 적용되도록 구조화
    * **Fail-fast 적용**: 잘못된 CTL 또는 설정으로 실행할 경우 즉시 오류를 반환하여 운영 실수를 조기에 발견하도록 구성
